# ねこ習慣ノート

「目標達成 × 猫育成」をテーマにした、スマートフォン向け習慣化Webアプリです。毎日の行動を記録すると、自宅に迷い込んできた茶トラ猫とのなかよし度が少しずつ上がります。猫は進化せず、好感度や継続状況によって表情・距離感・短いセリフが変化します。

## 主な機能

- 目標の設定・変更
- 行動内容、時間・回数、メモの簡単記録
- 好感度と猫の状態・ランダムセリフ
- 連続達成日数、合計記録日数、今日の記録状況
- 1日1回のログインボーナス
- キャットフード、おやつ、おもちゃのプレゼント
- 3日・7日・10日の未ログイン期間に応じた距離感の変化
- 最新5件の記録表示

10日以上空いて猫が出かけても、再び行動を記録すれば戻ってきます。ユーザーを責めたり、強いペナルティを与えたりしない設計です。

猫カードは縁側と小さな庭が見える和風の猫部屋UIです。好感度に応じて猫の距離感・状態ラベル・短いセリフが5段階で変化し、今日の記録状況と連続日数もカード内で確認できます。猫画像がなくても立体感のあるCSS・絵文字プレースホルダーで表示され、今後 `assets/cats/` に画像を追加できます。

猫画面は、左の小さな庭・中央の縁側・右の畳を見渡せる、横長ステージ型のオリジナル和風猫部屋です。背景から浮いて見えていた重ね置きの小物は一旦整理し、今後は背景になじむ画像素材で再追加する予定です。

猫には呼吸・顔を傾ける・目を細める雰囲気を組み合わせた控えめな待機モーションがあります。記録後とプレゼント後は、起き上がる、近づく、左右に喜ぶ短いリアクションが実画像猫とCSS猫の両方に適用されます。CSS猫ではしっぽ・耳・まばたきも維持します。`prefers-reduced-motion` に対応し、動きを減らす設定ではアニメーションを停止します。

プレゼントを渡すと猫カードへ自動で移動し、アイテムと好感度に応じた返答、ハート、弾む動き、やさしいカードの光で反応します。プレゼントは行動記録とは別の履歴として保存され、「最近のできごと」に表示されます。動きを減らす設定では自動スクロールと演出を最小限にします。

行動記録は1日に何度でも保存できますが、記録による好感度+3は1日1回だけです。プレゼントによる好感度上昇は所持数がある限り維持されます。ミドリの好感度が100になると、一度だけ黒猫の「クロ」が同じ猫部屋へやってきます。猫ステージと猫一覧データは複数猫に対応し、将来は同じ部屋に猫10種類を増やし、猫ごとの好感度を管理できる構造です。

## 使用技術

- HTML
- CSS
- JavaScript
- Web Storage API（localStorage）

外部API、外部画像、フレームワークは使用していません。

## 猫画像

`assets/cats/cat-main.png` にミドリ、`assets/cats/cat-kuro.png` にクロのPNGを配置すると猫画像を表示します。猫画像は背景透過PNGを推奨します。画像がない猫は、茶トラ・黒猫それぞれのCSSプレースホルダーへ自動で切り替わります。詳細は [assets/cats/README.md](assets/cats/README.md) を参照してください。

## 猫部屋・アイテム画像

`assets/room/cat-room-bg.png` に横長のPNG背景を配置すると、CSS背景から自動で切り替わります。同じファイル名で差し替え可能です。画像がない場合や読み込めない場合も、CSSの庭・縁側・和室が表示されます。背景の推奨仕様は [assets/room/README.md](assets/room/README.md) を参照してください。将来の猫画像とアイテム画像は、それぞれ `assets/cats/`、`assets/items/` へ追加する方針です。

## GitHub Pages

静的ファイルだけで動作します。GitHub Pages の公開元を `main` ブランチのルートに設定して公開する想定です。

## データ保存の注意

目標、記録、好感度、所持アイテム、最終ログイン日はブラウザの `localStorage` に保存され、外部には送信されません。ブラウザデータの消去や端末変更でデータは失われます。共有端末での利用には注意してください。
